# Screenshots guide

Estas son las capturas mas utiles para un portfolio tecnico o una demo compartida:

1. README principal abierto en el repo
2. Conversacion resuelta con fuentes limpias
3. Caso ambiguo que fuerza aclaracion
4. Confirmacion de ticket creado
5. Reporte de evaluacion RAG en Markdown
6. Diagrama de arquitectura
7. Teams custom app instalada en modo personal

## Recomendaciones practicas

- usa datos demo coherentes con el corpus
- evita mostrar secretos, `.env` o pantallas de Azure con credenciales visibles
- si capturas Teams, incluye una respuesta con card y otra con fallback textual
- si capturas evaluacion, intenta mostrar summary y worst cases en la misma imagen
